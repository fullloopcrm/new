  just missed here").

  gap.

  (`.eq`/`.in`/`.neq`), no raw SQL string-building; `target_url` fed to
  `fetchTitleMeta` comes from stored GSC issue rows, not live user input.

  aggregate across ALL tenants' `seo_metrics` with no tenant filter — by
  itself a materialized-view RLS-bypass shape, but checked every call site
  (`admin/seo/route.ts`, `lib/selena/tools.ts`) and both correctly scope
  down to the calling tenant's properties before reading the matview
  (`selena/tools.ts` line ~1386: `.in('property', properties)` where
  `properties` is derived from a `.eq('tenant_id', tid)` lookup one step
  earlier). No cross-tenant leak reachable from a tenant-facing surface.
- `lib/seo/tenants/*.ts`: static per-tenant sitemap/route descriptors, no
  secrets, no user input.
