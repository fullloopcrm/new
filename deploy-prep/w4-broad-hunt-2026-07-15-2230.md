## Areas checked

1. **OAuth CSRF (state param)** — `google/auth`+`callback`,
   `admin/google/auth`+`callback`, `social/connect/facebook`+`callback`.
   All mint/verify a signed `oauth-state` (HMAC, tenant-bound) before
   exchanging any code. Instagram callback uses the same
   `verifyOAuthState` helper. No forgeable state anywhere in this flow.

2. **CORS wildcard usage** — only on the two already-public,
   already-rate-limited tracking endpoints (`api/track`, `api/leads/visits`
   OPTIONS preflight). Intentional (marketing pages call cross-origin),
   not a gap.

3. **Referral-code entropy / IDOR** — `POST /api/referrals` still
   generates the operator-created referral code via `Math.random()` (not
   crypto), but it's an admin-only (`referrals.create`-gated), publicly
   shared marketing code, not an auth credential — no privilege from
   guessing/colliding it. The actual unauthenticated PII/financial IDOR on
   this same code (`/api/referrers`, `/api/referral-commissions`) was a
   prior finding already fixed (confirmed via `git log`: 099a2e15,
   63c5c5e0, and others already ported this branch).

4. **CSV formula-injection, second code path** — `src/lib/finance-export.ts`
   (used by `/api/cpa/[token]/year-end-zip`) has its own `csvEscape()`,
   separate from `lib/csv.ts` fixed in an earlier pass. Confirmed it also
   neutralizes leading `=+-@\t\r`. Clean.

5. **CPA year-end-zip token route** — `randomBytes(24)` base64url token
   (192-bit), tenant/entity-ownership checked at mint time, expiry +
   revocation checked at read time, atomic usage-count bump. No rate limit
   on the zip-build itself, but the token space is unguessable so this
   isn't a practical DoS/enumeration vector — lower priority than the
   SMS/email/Stripe-session classes fixed earlier this session, left as-is.

6. **Regex construction from server-controlled vs. user-controlled input**
   (ReDoS / regex-injection sweep of all `new RegExp(...)` call sites in
   `src/app` + `src/lib`) — `bank-import.ts` interpolates a hardcoded tag
   name (never user input) into its OFX parser regex; `site-export.ts`
   escapes the host before interpolating; `selena*/core.ts` interpolates a
   dictionary keyword (not the untrusted chat text) into its match regex.
   No attacker-controlled regex metacharacters reach any `RegExp`
   constructor.

7. **Upload MIME allow-lists** — re-verified all 7 direct-upload routes
   (`cleaners`, `uploads`, `booking-notes`, `public-upload`,
   `team-applications`, `management-applications`, `admin/notes`) each
   enforce an explicit `ALLOWED_TYPES`/`OK_TYPES` array against
   `file.type`, storage `contentType` is set to that same checked value —
   no SVG/HTML in any allow-list, no unchecked route found.

8. **PIN brute-force** — `POST /api/team-portal/auth` already caps both
   per-IP (5/15min) and tenant-wide (30/15min) guess rates, closing the
   distributed-spray gap. `admin/users/[id]/pin` (the newer hashed-PIN
   system) enforces per-tenant uniqueness against `pin_hash`, not
   plaintext.
